Operating Systems

// definition · classification · file management · MS-DOS · Windows

01Definition

Definition

An Operating System is a set of programs used to manage the basic hardware resources of a computer.

  • It is the main program that controls the execution of user applications.
  • It enables the user to access the hardware and software resources of the computer.
  • When the computer is switched on, the OS runs and checks that all parts of the computer are functioning properly.

02OS Platform

Application programs do not use hardware devices directly. They send messages through the OS, which gives instructions to the hardware to perform tasks. The OS acts as a layer between the user and the bare hardware.

Layer 1

User / Application

User communicates intentions via commands. Applications run on top of the OS.

Layer 2

Operating System

Manages all hardware resources; acts as interface between user and hardware.

Layer 3

Hardware

The bare physical components — CPU, RAM, storage, I/O devices.

Supervisor / Kernel

The OS contains a special program called the Supervisor (Kernel/Executive) stored in ROM. It contains essential commands and controls all other OS programs. OS programs are too large to fit in RAM at once — they are stored on the hard disk and loaded into RAM when needed.

03Why an Operating System is Needed

  1. Modern computer systems are complex and fast — they need internal control.
  2. To ensure the full system software facilities are readily available.
  3. To control jobs in what they are allowed to do, for security.
  4. To increase throughput — the amount of data processed in a given time.
  5. To improve communication between the user and the computer.
  6. To make complex tasks simple for the user.
  7. To help the computer correct errors — diagnostic messages are displayed when errors occur.
  8. To reduce job setup time — while one job runs, others can be queued.
  9. To allow many programs and users to share the system simultaneously.

04Resources Under OS Control

As a resource manager, the OS manages the following:

Resource 1

Processor (CPU)

Controls which programs execute and for how long.

Resource 2

Main Memory (RAM)

Allocates and tracks memory usage by programs.

Resource 3

Secondary Storage

Manages data storage and retrieval on disks.

Resource 4

I/O Devices & Ports

Controls input/output devices and their ports.

Resource 5

Communication Devices

Manages network and communication ports.

Resource 6

Files

Organises, retrieves, shares and protects files.

05Functions of an Operating System

1. Processor Management

The CPU can only execute one program at a time. The OS arranges tasks by priority and can pause one task to allow the CPU to service another, preventing conflicts in a multi-tasking environment.

2. Memory Allocation & Loading

Programs must be loaded into RAM before execution. The OS tracks which parts of memory are in use, which are free, and manages data files used by executing programs.

3. I/O Device & Port Management

The OS controls input from and output to all devices, monitors device state, signals faults, and resolves conflicts when multiple programs request the same device.

4. Secondary Storage Management

The OS manages storage and retrieval of data on secondary devices and uses free disk space to temporarily hold tasks that are waiting to be processed (virtual memory).

5. Communication Device Management

Controls the communication process between tasks and the computer — internally between programs and externally through communication ports via cables or wireless media.

6. File Management

Concerned with the logical organisation of the file system — provides means to sort, retrieve, share, and protect data files and programs against unauthorised access and corruption.

7. Job Scheduling

Arranges and loads programs to provide a continuous sequence of processing. Allocates priorities to jobs to ensure the CPU never remains idle.

8. Job Sequencing

Keeps a list of jobs currently running and monitors them as they move in and out of the processor. Arranges them in order for efficient execution.

9. Resource Control & Allocation

Gives each resource a unique Interrupt Request Number (IRQ). When two tasks request the same resource, the one with a higher priority IRQ is granted access, preventing deadlock.

10. Error Reporting & Correction

Monitors the system and performs error checks on hardware and software. Prints diagnostic messages suggesting correction routines. Fatal errors (e.g., dividing by zero) cause the program to terminate.

11. Interrupt Handling

Definition

An interrupt is a break from the normal sequential processing of instructions. When an interrupt occurs, control passes to the Kernel, which determines the cause and coordinates recovery.

Common causes of interrupts:

  • Power failure — Kernel saves vital information using dying power supply.
  • Arithmetic/logic errors — ALU detects errors and signals an interrupt.
  • Hardware malfunction — parity errors trigger the Kernel via interrupt.

A device driver is software the OS uses to control a specific piece of hardware.

12. User–Hardware Interface

Provides quick communication between the user and the computer. The user issues commands; the OS responds via screen or printer — establishing a "conversation".

13. Logging & Accounting

Keeps internal records of how resources (CPU time, memory, peripherals) are used and maintains a complete log of all processing activity.

14. Protection

Protects hardware, software and data from improper use and ensures application programs use hardware efficiently.

07Characteristics of an Operating System

Trait A

Reliable

At least as reliable as the hardware; detects errors and minimises damage.

Trait B

Protected

Protects users from each other's errors and malicious tampering.

Trait C

Efficient

Minimises idle time of user resources; uses as little overhead as possible.

Trait D

Convenient

Flexible and easy to use; designed with the user in mind.

Trait E

Predictable

Service quality should not vary widely over time; users should get estimates of when tasks complete.

08Classification of Operating Systems

Operating systems can be classified by three criteria:

Criterion 1

Tasks Handled

  • Single-tasking
  • Multi-tasking
Criterion 2

Number of Users

  • Single-user
  • Multi-user
Criterion 3

Human-Computer Interface

  • Command line
  • Menu-driven
  • GUI

09Classification by Tasks

Single-tasking OS
  • Only one user-program in memory at a time
  • User must exit a program before loading another
  • Example: MS-DOS
Multi-tasking (Multiprogramming) OS
  • Multiple programs in memory simultaneously
  • Each program gets a time-slice
  • CPU switches rapidly between programs
  • Appears to run programs simultaneously
  • Examples: Windows, Linux, UNIX

10Classification by Number of Users

Single-user OS
  • Only one person can use the computer at a time
  • Runs one user program at a time
  • Example: MS-DOS
Multi-user OS
  • Many users can access the computer simultaneously
  • Examples: UNIX, Novell Netware, Windows 2000/NT, Linux

11Classification by Human-Computer Interface (HCI)

HCI refers to the method of interaction between the user and the computer. A good HCI should be user-friendly: easy to learn, self-contained, robust, and logical.

1. Command-Driven Interface

The user types commands at a prompt. The computer reads and executes them. More flexible but harder to learn — preferred by experienced users.

MS-DOS Command Line
C:\> COPY Fruits.Dat A:\
// Copies Fruits.Dat from drive C to floppy drive A

C:\> PRINT Report1 Report2 Report3
// Prints three documents on the default printer

Disadvantages of Command-Driven Interfaces

  • More difficult to learn — user must know the exact command and syntax
  • Less user-friendly
  • Not suitable for beginners

Examples: MS-DOS, early PC-DOS, OS/2, UNIX

2. Menu-Driven Interface

Provides a list of commands on screen for the user to choose from. Suitable for beginners and infrequent users.

Types of Menus
  • Pull-down menus — common in Windows
  • Pop-up / Pop-down menus — appear above or below an item
Advantages
  • Many options to select from
  • No need to memorise commands
  • Easier to use than command-driven

3. Graphical User Interface (GUI)

Commands are issued by clicking on icons (small graphic images) using a pointing device like a mouse. Found on workstations and PCs with graphic adapters.

GUI Feature

Icons

Programs represented graphically as small images

GUI Feature

Pointing Device

Mouse or trackball used to select and issue commands

GUI Feature

Pull-down Menus

Menus that appear when you click a menu item

GUI Feature

Windows

Programs open by displaying rectangular windows on screen

Advantages of GUI

  • User-friendly
  • Easy to learn and use

Examples: Windows, Linux (with desktop), Apple Macintosh, OS/2 Presentation Manager

12Factors for Choosing an Operating System

  1. Hardware configuration — RAM size, hard disk capacity, processor type
  2. Basic computer design — IBM-compatible or Apple?
  3. Hardware compatibility
  4. User needs — the intended applications
  5. User-friendliness / HCI — command line, menu-driven, or GUI?
  6. Availability — e.g., Windows-based OS are very common
  7. Portability
  8. Cost
  9. Reliability — does it crash or hang?
  10. Communication method — number of peripherals supported
  11. Method of operating the computer

13File Management — Files & Directories

Most microcomputers store data on disks in files — groups of related records. The OS maintains a directory — a list of filenames and their disk addresses.

MS-DOS Filename Rules

  • Filename must not exceed 8 characters
  • Extension is 3 characters after a period (e.g., COMMAND.COM)
  • Common extensions: .COM and .EXE for program files, .TXT for text files

Example Directory Listing (Drive A:)

DIR A:
Volume in drive A: has no label
Directory of A:

COMMAND COM 18107 13-12-97 13:00PM
SYS COM 7407 13-12-97 13:00PM
FIND EXE 1278 13-12-97 13:00PM

4 FILE(S) 142890 BYTES FREE

The directory listing shows: filename, extension, size in bytes, date and time last saved, and bytes remaining on disk.

14Directory Hierarchy

Files on a disk can be organised into subdirectories forming a tree-like structure: root directory → subdirectories → files.

Directory Tree Example
KENYA\ ← root directory
├── NAIROBI\
│ └── HURLINGHAM
├── MOMBASA\
│ └── NYALI
└── NAKURU\
└── MILIMANI

// Path to Milimani: KENYA\NAKURU\MILIMANI

A path is a logical, sequential list of directories leading to a filename. Directory names and subdirectory names are separated by a backslash \ (or hyphen/slash in some DOS versions). The topmost directory is called the root directory.

15Volume Labels and Directories

A volume label is a name given to a disk to identify its contents when it is not mounted on a drive. It is recorded on the disk itself and checked by the OS to confirm the correct disk is installed.

Volume Label Example
// If volume label is COUNTRY, the full path to Milimani is:
COUNTRY\KENYA\NAKURU\MILIMANI

16Tab Control Language (TCL)

The user communicates with the OS through Tab Control Language (TCL) — a set of OS commands (system commands) that control the operating system.

When the OS is ready for commands, it displays a prompt. In MS-DOS this is:

MS-DOS Prompt
C:\> ← "C" = logged drive (active drive)
A:\> ← drive A is now the active/logged drive

The drive identifier (e.g., C:, A:, B:) must always include the colon.

17MS-DOS Command Syntax

A DOS command has a command name and arguments separated by spaces.

Syntax
// Format: COMMAND argument1 argument2

C:\> COPY ANN1 ANN2
// COPY = command name | ANN1, ANN2 = arguments
// Copies file ANN1 and saves the copy as ANN2

// To run a program (e.g., BASIC.COM):
C:\> BASIC ← type filename without extension

DIR Command Switches

DIR Examples
C:\> DIR ← list all files
C:\> DIR/P ← pause after each screen
C:\> DIR/W ← wide (5 columns, names only)
C:\> DIR B:FILENAME.TXT ← show specific file

C:\> DIR Same*.DBF ← SAME files with .DBF extension
Wildcards

* (asterisk) — substitutes any number of characters (e.g., *.BAS matches all .BAS files).
? (question mark) — substitutes exactly one character (e.g., ?AME matches NAME, SAME, etc.).

20Disk Management Commands

Formatting a Diskette

Formatting prepares a new diskette for data storage — organises the disk, keeps track of files, and marks any defective areas.

CommandDescription
FORMATFormat a data disk (no OS)
FORMAT B:Format the disk in drive B
FORMAT /SFormat and include the OS (creates a boot/system disk)
FORMAT /S B:Format drive B with OS
SYS B:Copy OS hidden files to a formatted disk in B
COPY COMMAND.COM B:Copy COMMAND.COM to the disk in B (step 2 of adding DOS)
LABELAssign a volume label (up to 11 characters) to a disk
DISKCOPY A: B:Copy entire contents of drive A to drive B (includes OS and formatting)
FORMAT output
C:\> FORMAT B:
INSERT NEW DISKETTE FOR DRIVE B: and strike any key when ready.
Formatting...
Format Complete
362496 bytes total disk space
362496 bytes available on disk
Format another (Y/N)?
DISKCOPY vs COPY

DISKCOPY A: B: copies everything including OS and formats the destination — but does NOT check for errors on the destination disk.
COPY A:*.* B: copies only files, checks for faulty areas on the destination and marks them — safer for non-full disks.

21Windows Operating Systems

Windows was developed as a GUI for PCs running MS-DOS. It allows users to enter commands by pointing and clicking on objects using a mouse. Takes its name from on-screen "windows" — rectangular areas set aside for specific tasks.

Examples of Windows Operating Systems

Legacy

Windows 95/98/ME

Early consumer versions with GUI and plug-and-play support.

Enterprise

Windows 2000/NT

Business-oriented, multi-user versions with networking support.

Modern

Windows XP/Vista

Improved stability, long filename support (up to 255 characters).

Open Source

Linux / UNIX / macOS

Also classified as Windows-like GUI operating systems.

Advantages of Windows OS

  • Easy to open programs, find documents, and switch between programs
  • Windows Explorer for easy file, folder, and drive management
  • Supports long file names (up to 255 characters including spaces)
  • Improved video, CD and sound capabilities
  • Plug and Play (PnP) — automatically recognises and sets up new hardware
  • True multi-tasking — run multiple programs simultaneously, each in its own window
  • User-friendly — menu driven with options shown as icons
  • Supports a wide choice of networks and transmission protocols
  • Electronic communications support — messages, file transfer, video calls
  • My Briefcase for file synchronisation and file transfers via cable/dial-up
  • All Windows applications share common commands: Open, Save, Print, etc.
  • Easy information exchange between Windows applications (cut/copy/paste)
  • Power-saving features — standby and hibernation
  • Built-in Help system with access to technical support via WWW
