Data Representation in a Computer

Complete Notes — Number Systems, Coding Schemes & Binary Arithmetic

01Introduction

  • Computers are classified according to functionality, physical size, and purpose.
  • By functionality: computers can be analog, digital, or hybrid.
  • Digital computers process data in discrete form; analog computers process data that is continuous in nature.
  • Hybrid computers can process both discrete and continuous data.
  • In digital computers, user input is first converted and transmitted as electrical pulses represented by two distinct digits: 1 and 0.
  • These two digits are called binary digits or, in short, bits.
  • Electronic signals that repeat at equal time intervals are called periodic signals.

A periodic wave can be described using three parameters:

Parameter A

Amplitude

  • The maximum value a wave can attain
  • E.g., amplitude of a standard signal wave = 1
Parameter f

Frequency

  • Number of cycles per second
  • Measured in Hertz (Hz)
  • 1 Hz = 1 cycle/second
Parameter T

Periodic Time

  • Time taken to complete one cycle
  • Formula: T = 1/f

02Modulation & Demodulation

When a digital signal is sent over analog telephone lines (e.g., email), it must be converted to an analog signal. This is done by connecting a modem to the digital computer.

Modulation
  • Converting a digital signal to an analog signal for transmission
  • Done at the sending end
Demodulation
  • Converting the incoming analog signal back to digital form
  • Done at the receiving end

03Data Representation in Electronic Circuits

Electronic components such as microprocessors are made of millions of electronic circuits.

State 1 (HIGH voltage / ON)
  • High voltage present in the circuit
  • Interpreted as binary 1
  • Analogous to a closed switch — bulb lights up
State 0 (LOW voltage / OFF)
  • Low voltage in the circuit
  • Interpreted as binary 0
  • Analogous to an open switch — bulb goes off

04Data Representation on Magnetic Media

Storage devices are coated with special magnetic materials such as iron oxide.

Binary 1
  • Magnetic field present in one direction
  • Dipoles of iron oxide particles face the same direction
Binary 0
  • Magnetic field present in the opposite direction
  • Dipoles arranged in the opposite direction

Data is written by arranging the magnetic dipoles of iron oxide particles so that some face one direction and others face the opposite direction. The dipoles are arranged in groups on the track of the disk.

05Data Representation on Optical Media

Optical devices (e.g., CD-ROM) use light to represent data. The shiny surface of a CD has very tiny holes called pits; areas without pits are called land.

Binary 1 — Land
  • Laser beam reflects from the flat land
  • Reflected light → interpreted as 1
Binary 0 — Pit
  • Laser beam enters a pit and is not reflected
  • No reflection → interpreted as 0

The reflected pattern of light from the rotating disk falls on a photoelectric detector that transforms the patterns into digital form.

06Why Binary is Used in Computers

  • It has proved difficult to develop devices that can understand or process natural language due to its complexity.
  • It is, however, possible to develop devices that understand binary language.
  • Binary logic has simplified the technology needed to develop both hardware and software systems.
  • Digital devices are more reliable, smaller in size, and use less energy compared to analog devices.

08Number Systems

Higher number systems are used in computing to reduce long streams of binary digits into manageable form, improving processing speed and optimising memory usage. The four major number systems in computing are:

Base 10

Decimal

  • Digits: 0–9
  • Most common in everyday life
Base 2

Binary

  • Digits: 0, 1
  • Native language of computers
Base 8

Octal

  • Digits: 0–7
  • Shorthand for binary groups of 3
Base 16

Hexadecimal

  • Digits: 0–9, A–F
  • Shorthand for binary groups of 4

09Decimal Number System (Base 10)

The term decimal is derived from the Latin prefix deci meaning ten. It uses digits 0–9 and is also called the denary or base-10 system.

Three Parameters of a Number

  • Absolute value — the magnitude of a digit itself (e.g., digit 5 in 7458 has an absolute value of 5).
  • Place value (positional value) — the position of the digit in the number (ones, tens, hundreds, thousands, etc.).
  • Base value (radix) — depends on the number system in use.

13Binary ↔ Decimal Conversions

Binary → Decimal

METHOD: Place Value
  1. Write the place values starting from the right-hand side (2⁰, 2¹, 2², …).
  2. Write each binary digit under its corresponding place value.
  3. Multiply each digit by its place value.
  4. Add up all products — the result is the decimal equivalent.

Decimal → Binary

Method 1: Long Division
  • Continuously divide the decimal number by 2.
  • At each step, write the remainder (0 or 1) to the right.
  • Read the remainders from bottom to top — this gives the binary equivalent.
Method 2: Place Value
  • Write down place values in factors of 2 up to the value ≥ the number.
  • Subtract each place value from left to right.
  • If difference ≥ 0: write 1; carry the difference forward.
  • If difference < 0: write 0; carry the number forward.

Example: 247₁₀ → Binary (Place Value Method)

247₁₀ = 011110111₂ = 11110111₂

14Binary Fractions

Binary Fraction → Decimal

For binary numbers with a fractional part, the place values to the right of the decimal point decrease as negative powers of 2 (2⁻¹ = 0.5, 2⁻² = 0.25, 2⁻³ = 0.125 …).

Example: 11.11011₂ → Decimal

2¹2⁰.2⁻¹2⁻²2⁻³2⁻⁴2⁻⁵
11.11011
21.0.50.2500.06250.03125
Integral: 2 + 1 = 3
Fractional: 0.5 + 0.25 + 0 + 0.0625 + 0.03125 = 0.84375
Total = 3.84375₁₀

Decimal Fraction → Binary

METHOD
  1. Convert the integer part using long division or place value method.
  2. For the fractional part: multiply it by 2 and note the product.
  3. Take the fractional part of that product and multiply by 2 again.
  4. Continue until the fractional part becomes 0 or starts repeating.
  5. Read the integer digits of each product from top to bottom — this gives the binary fractional equivalent.
  6. Combine both parts.
NB

When converting a real number from binary to decimal, work out the integral and fractional parts separately, then combine them.

17Symbolic Representation Using Coding Schemes

In computing, a single character (letter, number, or symbol) is represented by a group of bits. The number of bits per character depends on the coding scheme used.

4-bit / 6-bit

BCD

  • Binary Coded Decimal
  • 4-bit form: numeric data only
  • Standard BCD: 6-bit, represents 64 characters (including non-numeric)
  • Used in calculators, microwave ovens (LCD displays)
  • E.g., digit 9 = 1001; letter A = 110001
8-bit

EBCDIC

  • Extended Binary Coded Decimal Interchange Code
  • 8-bit scheme → 256 (2⁸) characters
  • Used primarily on IBM computers
  • E.g., letter A = 11000001
7-bit / 8-bit

ASCII

  • American Standard Code for Information Interchange
  • 7-bit: 128 (2⁷) characters
  • 8-bit extended: 256 characters
  • E.g., letter A = 1000001

18Representation of Signed Binary Numbers

There are three common ways to represent signed binary numbers in computers:

1. Extra Sign Bit

Prefix a digit to the binary number: 0 for positive, 1 for negative.

+127 = (0) 1111111₂
−127 = (1) 1111111₂
Problem: zero can be represented two ways — (0)0000000 and (1)0000000

2. One's Complement

Apply a bitwise NOT — flip every bit (0 becomes 1, 1 becomes 0). This is also called logical negation.

1100₂ → ones complement → 0011₂
00101101₂ → ones complement → 11010010₂ (represents −45₁₀)
Problem: zero still has two representations (+0 and −0)

3. Two's Complement (Most Popular)

Obtain the one's complement, then add 1. This is the standard method used in modern computers.

Advantages
  • Only one representation of zero
  • Addition and subtraction work correctly without extra circuitry to check the sign
Method
  • Get the binary equivalent
  • Find one's complement (flip all bits)
  • Add 1 to the result
// Two's complement of 45₁₀
45₁₀ = 00101101₂
One's complement: 11010010₂
Add 1: + 00000001
Two's complement: 11010011₂

Subtraction Using One's Complement

The computer performs binary subtraction by adding the one's complement of the subtrahend.

// 5 − 3 using one's complement (8-bit)
Rewrite as: 5 + (−3)
3₁₀ = 00000011₂
One's complement of 3: 11111100₂

  00000101 (5)
+ 11111100 (−3 in one's complement)
──────────
(1)00000001 ← overflow bit + 8-bit result

// Add the overflow bit back to the result:
00000001 + 1 = 00000010₂ = 2₁₀ ✓
One's Complement Rule

When using one's complement, the overflow (carry-out) bit is added back to the 8-bit result to obtain the correct answer.

Subtraction Using Two's Complement

Like one's complement, but after finding the one's complement, add 1 before the final addition. The overflow bit is then simply discarded.

// 5 − 3 using two's complement (8-bit)
Rewrite as: 5 + (−3)
3₁₀ = 00000011₂
One's complement: 11111100₂
Add 1 (two's comp): 11111101₂

  00000101 (5)
+ 11111101 (−3 in two's complement)
──────────
(1)00000010 ← overflow bit discarded

// Result (ignoring overflow):
00000010₂ = 2₁₀ ✓
Two's Complement Rule

When using two's complement subtraction, the overflow (carry-out) bit is simply ignored. The remaining 8 bits give the correct answer directly.
