01 Introduction

Human beings have evolved from the Stone Age to a highly sophisticated and advanced society by inventing things like the wheel, fire, transistors, and today's ultra-modern devices like computers. The idea of computers started way back in the nineteenth century.

The first generation computers — called Electronic Numeric Integrator and Calculator (ENIAC 1) — were operated by plugging wires into a control panel that resembles the old telephone switchboards.

Computer Programming

A computer works by executing a set of instructions known as a program. The term programming refers to the process of developing computer instructions (programs) used to solve a particular task. It involves use of special characters, signs and symbols found in a particular programming language to create computer instructions.

A programming language is a special set of symbols that can be translated into machine-readable form by the computer when arranged in a particular sequence or order. Each language has a special sequence or order of writing characters, usually referred to as syntax.

Historical Note It was John von Neumann of Princeton University (USA) who first proposed storing programs in computer memory — one of the most dramatic developments in computer history. This not only increased processing speed but also allowed easy and flexible methods of editing and updating the program.

02 Description of Terms

Source Program

The source program refers to the program code that the programmer enters in the program editor window — not yet translated into machine-readable form. The source program is usually created using a particular programming language.

Object Code

Object code refers to program code in machine-readable form. A source code that is not in machine-readable form must be translated into object code.

Translators

A translator is a language processor — such as an assembler, interpreter, or compiler — that converts the source program into object code.

Assembler

An assembler translates assembly language into machine language that the computer can understand and execute.

Interpreter

An interpreter translates the source program line-by-line, allowing the CPU to execute one line before translating the next. The translated line is not stored in memory — meaning every time the program is needed, it must be translated again.

Compiler

A compiler translates the entire source program into object code at once. The object code can then be linked into a fully executable program (e.g. .EXE files like WINWORD.EXE).

03 Levels of Programming Languages

Programming languages are classified into two major levels:

  1. Low-level languages
  2. High-level languages

These levels are further subdivided into five generations. The 1st and 2nd generations consist of low-level languages; the 3rd–5th generations consist of high-level languages.

Low-Level Languages

Low-level languages are so named because the computer can understand them directly, or they require little translation effort. They are hardware-oriented and not portable — a program written for one computer cannot run on another.

Machine Languages (1st Generation)

Instructions are written in binary logic (0s and 1s). Many lines of code are needed even for simple tasks. Different CPUs have different machine codes, making them hard to understand for programmers but easy to execute for computers.

Assembly Languages (2nd Generation)

Developed to overcome the difficulties of machine language. Programmers write programs as symbolic operation codes called mnemonics (two or three letter abbreviations). An assembler converts them to machine language. Like machine language, assembly languages are machine-dependent.

High-Level Languages

High-level languages are close to human language (English-like) and can be read even by non-experts. They are machine independent, allowing the programmer to focus on problem-solving rather than machine operation.

High-level languages are classified into five groups:

Third Generation Languages (3GLs) — Structured/Procedural

Allow programs to be broken into modules called structured programming. Structured programming uses sequence, selection, and iteration.

Examples:

  • Pascal — teaching structured programming
  • FORTRAN — mathematics, science, engineering
  • COBOL — business data processing
  • BASIC — beginners; microcomputer users
  • C — system software; powerful yet readable
  • Ada — military, industrial, real-time systems

Sample Pascal Program — Straight Line Equation (Y = MX + C)

Program Straight_Line (Input, Output);
Var
  y, m, x, c: Integer;
Begin
  Writeln ('Input the value of m');  Readln (m);
  Writeln ('Input the value of x');  Readln (x);
  Writeln ('Input the value of c');  Readln (c);
  y := m * x + c;
  Writeln ('The value of y is:', y);
End.

Fourth Generation Languages (4GLs)

Provide more programming tools (forms, command buttons, application generators). The programmer selects graphical objects on screen. Examples: Visual Basic Delphi Pascal Visual COBOL.

Fifth Generation Languages (5GLs)

Designed around solving problems using human-like intelligence (AI). The computer solves the problem for the programmer. Examples: PROLOG Mercury LISP OCCAM.

Object-Oriented Programming (OOP)

Views a program as objects interacting with each other. Each object has state (data) and behaviour (functions). The bundling of data and functions within an object is called encapsulation. Examples: Simula C++ Java SmallTalk.

04 Advantages & Disadvantages

Low-Level Languages

✓ Advantages
  • CPU understands machine language directly — no translation needed
  • Executes faster — complex instructions are pre-broken into simple ones
  • Stable — rarely crash once written
✗ Disadvantages
  • Difficult and cumbersome to use and learn
  • Require highly trained experts to develop and maintain
  • Debugging is very difficult
  • Machine-dependent — not portable across hardware/software platforms

High-Level Languages

✓ Advantages
  • Portable — transferable from one computer to another
  • User-friendly and easy to learn
  • Flexible — enhances creativity and productivity
  • Much easier to debug (correct errors)
✗ Disadvantages
  • Many instructions per statement causes slower processing
  • Must be interpreted or compiled before execution

05 Program Development

Program development can be broken into the following stages:

  1. Problem Recognition — Understanding and interpreting the problem. Identify key words such as compute, evaluate, compare. Recognise undesirable situations, opportunities to improve, or new directives.
  2. Problem Definition — Determine likely input, processing activities, and expected output (IPO analysis). Define the boundaries of the expected program. Example: for calculating the area of a circle, Input = π & radius; Process = π × r²; Output = Area.
  3. Program Design — Develop the algorithm (the processing logic). Programs are broken into modules, each performing a specific task (modular programming).
  4. Program Coding — Convert the design model into actual program code using a programming language. The result is a source program.
  5. Program Testing & Debugging — Test the program and correct errors (bugs). Two types of errors:
    • Syntax errors — improper use of language rules; detectable by the translator
    • Logical errors — program runs but gives wrong output (runtime/execution errors); not detectable by the translator
    Methods: Desk checking (dry-run), debugging utilities, and test data.
  6. Implementation & Maintenance — Actual delivery and installation of the program. Includes review, maintenance, and documentation (internal comments + external user manuals).
Documentation Documentation should be done at all stages. Three target groups: User-oriented, Operator-oriented, and Programmer-oriented documentation.

06 Development of Algorithms

An algorithm is a limited number of logical steps that a program follows to solve a problem. Algorithms are most commonly expressed using pseudocodes and flowcharts.

Pseudocode

A pseudocode is a set of statements written in a readable, English-like language that expresses a program's processing logic. It is not executable by a computer.

Guidelines for a Good Pseudocode

  • Statements must be short, clear, and readable
  • Statements must be unambiguous (one meaning only)
  • Lines should be clearly outlined and indented
  • Show clearly the start, stop, and control structures
  • Input, output, and processing clearly stated using keywords: PRINT, READ, INPUT

Example 1 — Sum and Average of Two Numbers

START PRINT "Enter two numbers" INPUT X, Y SUM = X + Y AVERAGE = SUM / 2 PRINT SUM PRINT AVERAGE STOP

Example 2 — Area and Perimeter of a Rectangle

START PRINT "Enter length and width" READ L, W AREA = L * W PERIMETER = 2(L + W) PRINT AREA PRINT PERIMETER STOP

Example 3 — Classify People by Age

START PRINT "Enter the age" INPUT AGE IF AGE > 20 THEN PRINT "Adult" ELSE PRINT "Young person" ENDIF STOP

Program Flowcharts

A flowchart is constructed using standard shapes that have specific meanings. Symbols are joined by arrows showing the order of execution.

Standard Flowchart Symbols (ANSI)

Input / OutputParallelogramSpecifies an input (READ) or output (PRINT) operation
ProcessRectangleIndicates processing or data transformation (formula or sentence)

Guidelines for Drawing Flowcharts

  • One entry/starting point and one exit point
  • Clear, neat, and unambiguous
  • Use correct symbols at each stage
  • Avoid overlapping or crossing flow lines
  • Decision instructions should be simple YES/NO
  • Use connectors for long multi-page flowcharts
  • Verify the flowchart is logically correct and complete
✓ Advantages of Flowcharts
  • Quicker understanding of relationships
  • Effective synthesis — used as working models
  • Proper program documentation
  • Effective coding guide
  • Orderly debugging and testing
  • Efficient program maintenance
✗ Limitations of Flowcharts
  • Complex and clumsy for complex logic
  • Changes may require complete re-drawing
  • Symbols cannot be typed
  • No uniform drawing practice
  • Difficult to link conditions and actions

07 Program Control Structures

Control structures are blocks of statements that determine how statements are executed. There are three types in structured programming:

Sequence

The computer reads instructions from top to bottom, one-by-one — called sequential program execution. Each statement executes in order after the previous one completes.

Selection

Execution of statements depends on a boolean condition (true or false). Example boolean expression: x >= 20. The four types of selection controls are:

1. IF … THEN

Used when only one option is available. All other options are ignored.

IF <Condition> THEN Program statement (executed if condition is TRUE) ENDIF

Example: Reward students who attain 80% and above.

IF Mark > 80 THEN PRINT "Give reward" ENDIF

2. IF … THEN … ELSE

Used when there are two available options.

IF <Condition> THEN Statement 1 ← THEN part ELSE Statement 2 ← ELSE part ENDIF

Example: Football card decision.

IF Fault = Serious THEN PRINT "Give red card" ELSE PRINT "Give yellow card" ENDIF

3. Nested IF

Used when two or more conditions must be considered to make a selection.

IF <Condition 1> THEN Statement 1 ELSE IF <Condition 2> THEN Statement 2 ELSE IF <Condition 3> THEN Statement 3 ELSE Statement 4 ENDIF ENDIF ENDIF

Example: Olympic medals.

IF position = 1 THEN medal = "Gold" ELSE IF position = 2 THEN medal = "Silver" ELSE IF position = 3 THEN medal = "Bronze" ELSE medal = "None" ENDIF ENDIF ENDIF

4. CASE Selection

Alternative to Nested IF for multiple options; reduces lines of code. Boolean expression can only use integers or alphabetic characters.

CASE average OF 80..100 : Grade = 'A' 70..79 : Grade = 'B' 60..69 : Grade = 'C' 50..59 : Grade = 'D' 40..49 : Grade = 'E' ELSE Grade = 'F' ENDCASE

Iteration (Looping)

Iteration executes the same block of code repeatedly until a condition is fulfilled. Three types of loops exist:

Types of Loops

  • Conditional loop — Number of repetitions not known in advance
  • Unconditional loop — Repeated a specified number of times
  • Infinite/Continuous loop — Repeats without ending

The WHILE Loop

Condition is tested before executing statements. Statements may execute zero or many times.

WHILE <condition> DO statements ENDWHILE

Example: ATM cash withdrawal (requires positive balance).

WHILE balance > 0 DO Withdraw cash Update account END WHILE

The REPEAT … UNTIL Loop

Condition is tested after executing statements. Statements execute at least once.

REPEAT statements UNTIL <condition>

Example: ATM withdrawal (client can withdraw at least once).

REPEAT Withdraw cash Update account UNTIL balance > 0;

The FOR Loop

Used when statements must repeat a predetermined number of times.

FOR loop_variable = lower_limit TO upper_limit DO statements ENDFOR

Can also count downwards:

FOR loop_variable = upper_limit DOWN TO lower_limit DO statements ENDFOR

Example: Sum of ten numbers.

FOR count = 1 TO 10 DO PRINT "Enter a number (N)" INPUT N Sum = Sum + N END FOR Display SUM
How the FOR loop works 1. Loop variable (count) initialised to lower limit (1). 2. Lower limit tested against upper limit (10). 3. If count ≤ 10, execute body; else exit loop. 4. After body, count is incremented by 1. 5. Repeat from step 2.

08 Designing More Complex Algorithms

Example 1 — Division with Zero Check

Prompt user for two numbers X and Y. Divide X by Y, but if Y = 0, display an error message.

START PRINT "Enter 2 numbers X and Y" INPUT X, Y IF Y = 0 THEN PRINT "Error: Division by zero" ELSE Quotient = X / Y PRINT X, Y, Quotient ENDIF STOP

Example 2 — TSS Savings Interest Calculator

Pays 5% interest on shares > 100,000 and 3% on shares ≤ 100,000. No interest on deposits.

START PRINT "Enter member name, share and deposit" INPUT Name, Shares, Deposit IF Shares > 100000 THEN Interest = 0.05 * Shares ELSE Interest = 0.03 * Shares ENDIF TotalSavings = Deposit + Shares + Interest PRINT Name, TotalSavings, Interest STOP

Example 3 — Olympic Medals

1st = Gold, 2nd = Silver, 3rd = Bronze, otherwise = None.

START PRINT "Enter athlete Name and Position" INPUT Name, Position IF Position = 1 THEN Medal = "Gold" ELSE IF Position = 2 THEN Medal = "Silver" ELSE IF Position = 3 THEN Medal = "Bronze" ELSE Medal = "None" ENDIF ENDIF ENDIF PRINT Name, Medal STOP
